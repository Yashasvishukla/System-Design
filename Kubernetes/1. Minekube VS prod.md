
Kubernetes's job is to manage the resources and allocate them to the applications running on it.

## Nodes (Machines)

| Node | RAM |
|------|-----|
| Node 1 | 16GB |
| Node 2 | 8GB |
| Node 3 | 8GB |

## Pods (Applications)

| App | Required RAM |
|-----|--------------|
| App 1 | 12GB |
| App 2 | 2GB |
| App 3 | 5GB |
| App 4 | 4GB |
| App 5 | 4GB |

## Resource Allocation

Kubernetes looks at the resources required by each application and decides which node to run it on. It might do something like this:

| Node | Apps | RAM Left Over |
|------|------|---------------|
| Node 1 | App 1 (12GB), App 2 (2GB) | 2GB |
| Node 2 | App 4 (4GB), App 5 (4GB) | 0GB |
| Node 3 | App 3 (5GB) | 3GB |

## Why this won't work with Minikube
with minikube, we only get one node! So once our machine runs out of the resouces. We are out of luck. That's why minikuve is great for learning. but not for production.
