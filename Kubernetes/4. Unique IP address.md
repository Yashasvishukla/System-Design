## Unique Ip Addresses
Every pod in kubernetes cluster has a unique internal-to-k8s IP address.

By giving each pod a unique Ip, Kubernetes simplifies communication and service discovery within a cluster.

Pods within the same NODE or across different NODES can easily communicate.

All the resouces inside a k8s cluster are virtualized. So the IP address is not the same as the IP address of the NODE it's running on.
Its a virtual IP address that is only accesible from within the cluster.

Run this command to get a "wide" output of your pods:

`kubectl get pods -o wide`

It gives a few more columns of information, including the IP address of each Pod. Notice that each Pod has a unique IP address!

`kubectl proxy`

This will start a proxy server on your local machine. probably on 127.0.0.1:8001

The kubectl proxy command creates a gateway between your local machine and the Kubernetes API server.

Normally, the Kubernetes API server is secured and requires specific authentication (like certificates or tokens) to access. By running kubectl proxy, you are telling kubectl to:

Use your local credentials to authenticate with the remote cluster.
Open a local web server (usually on port 8001).
Forward any requests sent to that local port directly to the API server.
This allows you to browse your cluster's data using simple tools like a web browser or curl without having to manually manage headers or authentication for every request.
